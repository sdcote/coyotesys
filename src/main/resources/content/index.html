Products

Whether running on embedded SBCs in the field, hosted in the cloud or your own data center, Coyote Systems has the data integration solution components to meet your needs. Chose only the toolkits needed for your particular integration.

CoyoteDX

Core data exchange toolkit and framework for creating any number of managed integrations.

CoyoteFT

File Transfer tools supporting the transfer of files across FTP and SFTP. Great for pre and post processing tasks to stage data and deliver results.

CoyoteSN

Send and receive data from any ServiceNow instance. Uses the table model API to access any data in the SaaS instance and exchange it with any other system.

CoyoteUI

Allows for the support of an optional Web UI to directly manage the data exchange service and all the jobs it supports.

CoyoteMT

Send and receive data through SMTP, IMAP and POP3 services. Tools include components for sending emails on events in the data exchange jobs.

CoyoteWS

Send and receive data through any JSON/XML web service. Tools include support for SOAP and even screen scraping.

CoyoteSNMP

Read and write SNMP traps and events. Makes anything SNMP accessable and allows for automated monitoring and management of SNMP components. Allows an integration to not only communicate with SNMP devices, but to be monitored an managed by enterprise NMS.

CoyoteMQTT

Send and receive MQTT messages to any standardized broker. Supports message caching and SSL/TLS connections. Perfect for field deployment in embedded SBCs to communicate with head end or SCADA systems.

CoyoteDF

Data Factory tools to generate randomized data using any number of stratagies. Great for Load/Stress testing in CI pipelines or general exploratory testing.

CoyoteMB

Highly resilient components communicate with a peer to peer (serverless) packet protocol to coordinate work, monitor peers and enable OA&M capabilities for any application on any platform, from SBC to cloud deployments.

Device Convergence

Mediating the data models, communications protocols and operation activities of devices from many different vendors is challenging and complicated. Use device mediation to bring different devices from different vendors under one common model.

Service providers find themselves with a variety of devices deployed in the field. Not only different devices, but different vendors and different versions and models from the same vendor. It seems every device has its own protocol, data model, API and exchange pattern. The challenge is how to represent the different devices and their data in one unified model.

A successful approach is the use of device mediation components to abstract the vendor-specific details away from the system. CoyoteDX lives between the field devices and the data center, communicating with the devices in their own native protocols and transforming that data into one the rest of the organization understands. Data passes through the Perimeter Firewall in the devices native protocols but enters the Application Firewall using a single, uniform protocol thanks to the mediation of the CoyoteDX components.

Dedicated modules are written to interact with either the device directly or an intermediate (head-end) system and exchange data to the rest of the system using a common model. Voice and Data provisioning does not have to embed vendor-specific logic to interact with the hardware. Distribution automation systems can use IEC CIM to interact with grid components using the device mediation modules as a facade to different hardware platforms.

The result is vendor lock-in becomes a thing of the past, as CoyoteDX allows any equivalent device from any vendor to be deployed where appropriate and even replace existing devices from other vendors. You can implement Demand Management with devices from multiple vendors and let CoyoteDX handle protocol conversions. Grid Sensors from any vendor can be deployed without having to change any of the existing business logic. CoyoteDX ensures each vendor sends the same set of events and metrics so te back office can concentrate on just one model,
